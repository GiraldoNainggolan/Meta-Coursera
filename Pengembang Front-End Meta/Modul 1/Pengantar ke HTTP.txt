Pernahkah Anda memperhatikan ikon kunci di samping URL di browser web Anda? Ini berarti bahwa HTTPS, versi aman HTTP sedang digunakan. HTTP adalah protokol operasional inti dari world wide web. Inilah yang memungkinkan browser web Anda berkomunikasi dengan server web yang menghosting situs web. HTTP adalah protokol komunikasi yang Anda gunakan setiap kali Anda menjelajah web. HTTP adalah singkatan dari Hypertext Transfer Protocol. Ini adalah protokol yang digunakan untuk mentransfer sumber daya web seperti dokumen HTML, gambar, gaya, dan file lainnya. HTTP adalah protokol berbasis respon permintaan. Browser web atau klien mengirimkan permintaan HTTP ke server dan server web mengirimkan respons HTTP kembali ke browser. Selanjutnya, mari kita mulai menjelajahi susunan permintaan HTTP. Permintaan HTTP terdiri dari metode, jalur, versi, dan header. Metode HTTP menjelaskan jenis tindakan yang harus dilakukan klien. Metode HTTP utama atau yang paling umum digunakan adalah, GET, POST, PUT, dan DELETE. Metode GET digunakan untuk mengambil informasi dari server yang diberikan. Permintaan POST digunakan untuk mengirim data ke server. Metode PUT memperbarui apa pun yang saat ini ada di server web dengan sesuatu yang lain. Metode DELETE menghapus sumber daya. Path adalah representasi dari mana sumber daya disimpan di server web, misalnya jika Anda ingin meminta gambar dari halaman di situs web, maka URL di bilah alamat harus berisi jalur lengkap ke file tertentu di server web, seperti contoh dot com, gambar garis miring maju, gambar garis miring maju titik jpg. Ada beberapa versi protokol HTTP. Saya tidak akan menjelajahi ini sekarang, tetapi saya ingin Anda menyadari bahwa Versi 1.1 dan 2.0 adalah yang paling banyak digunakan. Akhirnya, ada header. Header berisi informasi tambahan tentang permintaan dan klien yang membuat permintaan. Untuk metode permintaan tertentu, permintaan juga akan berisi isi konten yang dikirim klien. Sekarang, mari kita bahas beberapa detail tentang susunan respons HTTP. Respons HTTP mengikuti format yang mirip dengan format permintaan. Setelah header, respon opsional akan berisi badan pesan yang terdiri dari isi respon seperti dokumen HTML, file gambar dan sebagainya. Kode status HTTP yang terkandung dalam header menunjukkan apakah permintaan HTTP berhasil diselesaikan. Nilai kode berada dalam kisaran 100-599 dan dikelompokkan berdasarkan tujuan. Pesan status adalah representasi teks dari kode status. Selama penjelajahan web Anda, pernahkah Anda menemukan halaman yang menampilkan kesalahan 404 tidak ditemukan atau kesalahan 500? Server tidak merespons. Ini adalah kode status HTTP. Saya ingin menjelaskan secara singkat kepada Anda tentang kode status dan pengelompokannya. Ada lima kelompok kode status. Mereka dikelompokkan berdasarkan digit pertama dari nomor kesalahan. Informasi dikelompokkan 100-199. Tanggapan yang berhasil dikelompokkan dari 200-299. Pesan pengalihan adalah 300-399. Respons kesalahan klien berkisar dari 400-499, dan respons kesalahan server adalah 500-599. Respons informasi adalah tanggapan sementara yang dikirim oleh server. Tanggapan ini bersifat sementara sebelum respons yang sebenarnya. Respons informasi yang paling umum adalah 100 lanjutkan, yang menunjukkan bahwa klien web harus terus meminta atau mengabaikan respons jika permintaan sudah selesai. Tanggapan yang berhasil menunjukkan bahwa permintaan berhasil diproses oleh server web, dengan respons sukses yang paling umum adalah 200 OK. Anda menerima tanggapan ini setiap hari ketika Anda berhasil menerima konten dari situs web. Arti OK, tergantung pada metode HTTP. Jika metode ini GET, itu berarti bahwa sumber daya ditemukan dan termasuk dalam tubuh respon HTTP. Jika itu POST, itu berarti bahwa sumber daya berhasil ditransmisikan ke server web dan jika PUT, sumber daya berhasil ditransmisikan ke server web. Akhirnya, jika metodenya DELETE, itu berarti sumber daya telah dihapus. Respons pengalihan menunjukkan kepada klien web bahwa sumber daya yang diminta telah dipindahkan ke jalur yang berbeda. Kode respons yang paling umum digunakan adalah 301 dipindahkan secara permanen dan 302 ditemukan. Perbedaan antara pesan pengalihan 301 dan 302 adalah bahwa 302 menunjukkan pengalihan sementara.Sumber daya telah dipindahkan sementara. Ketika browser web menerima tanggapan ini, mereka akan secara otomatis mengirimkan permintaan untuk sumber daya di jalur baru. Tanggapan kesalahan klien menunjukkan bahwa permintaan berisi sintaks atau konten yang buruk dan tidak dapat diproses oleh server web. Kode yang paling umum digunakan adalah 400 digunakan ketika browser web atau klien mengirimkan data buruk ke server web, 401 digunakan untuk menunjukkan bahwa pengguna harus masuk ke akun sebelum permintaan dapat diproses, 403 digunakan untuk menunjukkan permintaan itu valid, tetapi server web menolak untuk memprosesnya. Ini sering digunakan untuk menunjukkan bahwa pengguna tidak memiliki izin yang cukup untuk menjalankan tindakan dalam aplikasi web, 404 digunakan untuk menunjukkan bahwa sumber daya permintaan tidak ditemukan di server web. Respons kesalahan server menunjukkan bahwa kegagalan terjadi pada server web saat mencoba memproses permintaan. Kode yang paling umum digunakan adalah kesalahan server internal 500, yang merupakan status kesalahan umum yang menunjukkan bahwa server gagal memproses permintaan. Ini digunakan untuk komunikasi yang aman antara dua komputer sehingga tidak ada orang lain yang dapat melihat informasi yang dikirim dan diterima. Hanya komputer lain yang dapat mengubah kode rahasia kembali ke konten aslinya. Jika orang lain melihat kodenya, itu tidak akan dapat dimengerti. Anda menggunakan HTTPS setiap hari.
