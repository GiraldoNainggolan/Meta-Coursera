Saya yakin Anda telah memperhatikan bahwa ketika Anda berbelanja online, situs web akan sering memberi Anda rekomendasi yang dipersonalisasi. Tetapi ketika teman Anda masuk ke situs web yang sama, rekomendasinya akan berbeda. Tetapi apakah Anda pernah bertanya-tanya bagaimana situs web beradaptasi dengan profil pengguna yang berbeda? Dalam video ini, Anda akan mempelajari perbedaan antara konten statis dan dinamis. Anda juga akan dapat membedakan antara peran server web dan server aplikasi. Sekarang, Anda tahu bahwa ketika Anda membuka situs web, server web mengirimkan konten situs web ke browser Anda. Konten bisa statis atau dinamis. Konten statis adalah file yang ditransfer server sama seperti disimpan di server web, seperti video atau gambar. Konten dinamis, di sisi lain, dihasilkan ketika permintaan HTTP dibuat. Misalnya, konten dapat dihasilkan berdasarkan masukan dari pengguna, atau ketika Anda mengunjungi situs web berita, itu akan didasarkan pada tanggal saat ini. Apa yang sebenarnya terjadi, adalah bahwa server web berkomunikasi dengan jenis server lain, yang disebut server aplikasi atau back-end. Server aplikasi menghasilkan konten dinamis yang dikirim server web kembali ke browser pengguna. Sekarang Anda tahu perbedaannya. Mari kita lihat contoh bagaimana situs web memperbarui konten statis dan dinamis. Katakanlah misalnya, Anda ingin menonton video di situs web, Anda mengklik tombol Putar dan permintaan dikirim ke server web. Server web merespons dengan mengirimkan file ke browser Anda. Sekarang, bagaimana ini dibandingkan dengan konten dinamis? Karena konten dinamis dihasilkan saat Anda menggunakan situs web, biasanya dibutuhkan waktu lebih lama untuk menghasilkan daripada yang diperlukan untuk mengirim kembali konten statis. Misalnya, ketika Anda masuk ke kursus ini, server web berkomunikasi dengan server aplikasi untuk memeriksa apakah Anda sebenarnya terdaftar. Server aplikasi mengonfirmasi pendaftaran Anda dan menentukan konten apa yang harus ditampilkan untuk profil Anda secara khusus. Server aplikasi melakukan pemrosesan yang lebih kompleks daripada server web. Misalnya, mereka harus menjalankan logika aplikasi, berkomunikasi dengan database, dan memeriksa izin. Untuk meningkatkan kinerja, server aplikasi yang berbeda memiliki tujuan khusus. Ada server aplikasi untuk setiap jenis konten di luar sana, mulai dari streaming musik hingga membuat blog Anda sendiri. Anda bahkan dapat membuat server aplikasi Anda sendiri, seperti membangun back-end untuk aplikasi web. Tapi tunggu dulu, Jika konten dinamis lebih lambat dihasilkan, bukankah itu masalah bagi situs web besar? Ya, itu. Server aplikasi biasanya memiliki kapasitas terbatas pada berapa banyak permintaan yang dapat mereka proses per detik. Tapi untungnya, di sinilah server web dapat membantu. Server web menggunakan proses yang disebut caching alih-alih menghasilkan konten secara dinamis untuk setiap permintaan. Caching berarti server web menyimpan salinan konten dinamis. Jika konten diminta lagi, server web dapat segera mengirim versi cache ini alih-alih meneruskan permintaan lagi ke server aplikasi. Pada permintaan pertama untuk konten dinamis, server web memeriksa apakah konten ada di cache. Jika tidak ada, konten diminta dari server aplikasi dan disimpan dalam cache. Server web kemudian mengirimkan kembali konten dinamis ke browser. Pada permintaan berikutnya, server web segera mengirimkan kembali konten yang disimpan dalam cache. Ini mengurangi jumlah konten dinamis yang harus dihasilkan oleh server aplikasi. Kemudian, setelah jangka waktu tertentu atau dengan interaksi pengguna berikutnya, server web memperbarui cache dengan konten terbaru. Sekarang Anda tahu perbedaan antara konten statis dan dinamis, serta perbedaan antara web dan server aplikasi. Dengan informasi baru ini, cobalah untuk mengidentifikasi dengan konten di situs web yang Anda kunjungi bersifat statis atau dinamis.
