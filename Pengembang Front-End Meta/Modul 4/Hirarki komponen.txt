Merencanakan aplikasi sebagai serangkaian komponen bisa jadi rumit pada awalnya, tetapi setelah melakukannya beberapa kali, itu akan menjadi diatur di alam. Kami telah belajar tentang teknologi Internet inti dan sekarang Anda tahu bahwa pustaka JavaScript di React memungkinkan Anda mengembangkan antarmuka pengguna lebih cepat. Bukankah keren bahwa Anda dapat mengembangkan antarmuka pengguna begitu cepat? Dalam video ini, Anda akan mempelajari cara kerja hirarki komponen di React. Setiap aplikasi React mengandung setidaknya satu komponen, komponen root atau komponen aplikasi. Komponen ditambahkan ke komponen aplikasi untuk membangun struktur pohon komponen yang membentuk aplikasi. Mari kita jelajahi contoh bagaimana aplikasi mungkin terstruktur sebagai komponen. Misalkan aplikasi Anda adalah daftar belanja barang yang perlu diambil dari toko kelontong. Di root kami memiliki komponen aplikasi. Komponen aplikasi memiliki dua komponen anak, bilah item baru dan daftar belanja. Item baru berdasarkan komponen memungkinkan kita untuk menambahkan item baru ke daftar. Komponen daftar belanja berisi item belanja, komponen anak untuk setiap item dalam daftar belanja. Meskipun item itu sendiri mungkin berbeda, seperti ayam atau mie, itu ditampilkan dengan cara yang sama seperti item lainnya. Oleh karena itu, Anda dapat menggunakan kembali komponen item belanja untuk menampilkan beberapa item. Saat pengguna melepas item, daftar akan memperbarui dan menghapus komponen anak item belanja yang sesuai dari komponen daftar belanja. Sekarang setelah Anda memahami hierarki komponen, mari kita jelajahi bagaimana sebuah situs web dapat dipecah menjadi beberapa komponen. Sebagai contoh, saya akan berbicara dengan Anda melalui situs blog khas. Biasanya ada judul, menu, dan bilah pencarian. Anda juga biasanya memiliki posting blog unggulan yang menampilkan lebih besar dari posting blog sebelumnya di bawahnya. Sekarang, mari kita membongkar hirarki komponen. Pada root hirarki komponen, Anda memiliki komponen aplikasi. Ini adalah seluruh halaman web. Komponen aplikasi harus memiliki komponen turunan, komponen Navbar untuk bilah navigasi dan komponen Page. Komponen Navbar berisi judul blog, tautan navigasi untuk situs web, bersama dengan komponen pencarian. Dalam komponen halaman, kami memiliki tiga komponen. Komponen fitur utama dan dua contoh komponen fitur kecil. Komponen fitur utama menampilkan ringkasan blog tentang posting blog tertentu. Dalam komponen fitur kecil, ada ringkasan blog dan komponen thumbnail untuk menampilkan gambar yang terkait dengan posting blog. Perhatikan bahwa komponen fitur kecil digunakan dua kali. Ini adalah bagian kode yang sama, tetapi untuk fitur kecil kedua, kode menggunakan properti yang berbeda. Itulah manfaat membangun situs web dengan komponen React, Anda dapat menggunakan kembali kode untuk beberapa bagian. Menggunakan serangkaian komponen untuk membangun aplikasi bisa jadi rumit pada awalnya, tetapi setelah Anda melakukannya beberapa kali, Anda akan menghargai betapa bergunanya itu. Semoga sukses dengan merencanakan hierarki komponen Anda di React.
